基础开源框架
——Open5GS
Open5GS
是一个开源项目，用于实现 
5G 
和 
4G LTE 
移动核心网络，它提供了完整的 
EPC
（
Evolved Packet Core
，
4G
分组核心网）和 
5GC
（
5G Core
，
5G
核心网）组件。让开发者和研究人员能够搭建和测试移动通信网络环境。它继承自
NextEPC
，且社区支持良好，尤其是对于实验室环境和小型商业部署非常合适。
EPC
，其特点是仅有分组域而无电路域、基于全
IP
结构、控制与承载分离、网络结构扁平化。
5GC
，其控制面采用服务化架构设计（
SBA
，
Service Based Architecture
），将网络功能（
NF
）进行了拆分，而所有的 
NF 
又都通过接口接入到系统中，
SBA 
设计的目标是以软件服务重构核心网，实现核心网软件化、灵活化、开放化、智慧化。

===== slide6.xml =====
基础开源框架
——Open5GS
 SBA
架构关键技术点：
（
1
）服务的提供通过生产者（
Producer
）与消费者（
Consumer
）之间的消息交互来达成。交互模式简化为两种：
Request-Response
和
Subscribe-Notify
。
（
2
）实现了服务的自动化注册和发现。
NF 
通过服务化接口，将自身的能力作为一种服务暴露到网络中，并被其他 
NF 
复用；
NF 
通过服务化接口的发现获取拥有所需 
NF 
服务的其它 
NF 
实例。这种注册和发现通过 
5G 
的新型网络功能 
NRF 
实现： 
NRF 
接收其它 
NF 
发来的服务注册信息，维护 
NF 
实例的相关信息和支持的服务信息； 
NRF 
接收其它 
NF 
发来的 
NF 
发现请求，返回对应的 
NF 
实例信息。
（
3
）采用统一服务化接口协议。目前定义的接口协议栈，从下往上在传输层采用 
TCP
，在应用层采用 
HTTP/2.0
，在序列化协议方面采用 
JSON
，接口描述语言采用
OpenAPI
 3.0
，
API
的设计方式采用 
RESTFul
。

IMS
，其核心是
SIP
         SIP Proxy Server
，
IMS
称为
CSCF
（
 Call Session Control Function
，呼叫会话控制功能
），并且又把它分成多个子系统，其中代理
CSCF
（
P-CSCF
）直接与终端交互，然后交给查询
CSCF
（
I-CSCF
），
I-CSCF
会查询
HSS
数据，对用户名和密码进行认证，查询是否欠费，开通或关闭的业务，根据接入
P-CSCF
判断你是不是漫游。
         整个呼叫过程，
P-CSCF
是不做复杂工作的，
P-CSCF
只负责接收
SIP
消息，它相当于对外联络点，然后
SIP
消息会到达
I-CSCF
，
I-CSCF
是运营商核心网络（就是运营商内部网络）的入口，它会根据

，
S-CSCF
是真正的
VoIP
里
Proxy Server
的角色，
S-CSCF

称为
AS
，另外独立成一个子系统）。
